Python中安装pandas出现问题总结

  1. 首页
  2. 文章分类
  3. 正文

1、安装pandas总报超时：这个方法一般都能解决问题

解决方法：pip install 包名-i http://pypi.douban.com/simple/ --trusted-host pypi.douban.com

这个是因为你下载的包不对 如下32位的可以改成64位

两种方式安装都是报同样的错误，失败截图：

 

失败原因：大致是因为有些包是用C来写的，需要依赖gcc或者vc编译器。

针对这个问题的解决方案，网上也有一些相关文章，但是看了后觉得颇为麻烦，所以改为离线安装，正好三种方式都尝试一下

 

1. 下载对应的.whl安装文件

下载地址：https://www.lfd.uci.edu/~gohlke/pythonlibs/

 

2. 将下载好的文件放到python\Scripts目录下

3. cd 到Scripts目录下执行pip install 文件名

4. 如果出现报错，例如：

 

失败原因：包的版本与python版本不匹配。以上34，35，36，37，27代表的都是对应的python版本，请各位同学选择合适版本。若还是不行，在cmd命令行执行：

 

import pip

print(pip.pep425tags.get_supported())    #获取

以我自己来说，我的电脑操作系统是win10，64位，python版本是3.4.4，所以接下来将我的.whl文件重命名为pandas-0.20.3-cp34-none-win_amd64.whl（将cp34m更改为none）

5. 如果继续报错：

删除64位文件，下载32位的文件，并重命名

例如：pandas-0.20.3-cp34-none-win32.whl

6. 继续2，3的步骤

7. 安装成功
